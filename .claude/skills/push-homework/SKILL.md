# 推送作业（push-homework）

将本地更改完整地推送到 GitHub 远程仓库。自动处理 Git 提交与推送的完整流程。

## 工作流程

1. **git add .** – 添加所有更改到暂存区
2. **git status** – 检查是否有变更需要提交
3. **git commit** – 生成带当前日期时间的提交信息，如 “作业提交 2026-04-15 14:30”
4. **git pull --rebase** – 先拉取远程最新代码（防止非快进拒绝）
5. **git push** – 推送到远程仓库

## 提示

- 如果没有任何更改，则不需要创建新的提交。
- 如果存在冲突，则提示解决后再继续推送。
